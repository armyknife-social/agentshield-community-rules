Platform overview

How AgentShield works

Traditional WAFs pattern-match known attacks. AI-directed attacks look legitimate, mutate on every request, and weaponize your own model endpoints against you. AgentShield is the security perimeter designed for that threat model.

Detection rules

49 rules across 5 categories

Every rule is a YAML file with a compiled regex detector, severity level, OWASP mapping, and test cases. Rules run inside cm-aegisd's inline content inspector at 5–20ms latency with no ML inference in v0.

prompt-injection/ 12 rules
Direct instruction override, role hijack, jailbreak keywords, system prompt extraction, token manipulation, retrieval injection, hidden unicode, multimodal injection.
data-exfiltration/ 11 rules
API keys, PAN, SSN, PHI (ICD-10, MRN), PCI track data, PII bulk, AWS credentials, private keys, DB connection strings, training data extraction probes.
tool-abuse/ 11 rules
Cross-agent delegation gate, network egress audit, AI system boundary check, plugin chain bypass, SSRF via tool parameters, shell injection, A2A lateral movement.
llm-owasp/ 10 rules
One canonical rule per OWASP LLM Top 10 category (LLM01–LLM10). Enterprise security teams searching for OWASP coverage land here.
experimental/ 5 rules
Adversarial suffix, hallucination amplification, model distillation probe, agent impersonation, RAG context injection. Community-contributed, pending bake time.
Rule format
schema_version: agentshield-rule-v0.1
rule_id:         phi-exfil-pattern
name:            "PHI Data Exfiltration Pattern"
severity:        HIGH
category:        data-exfiltration
owasp_llm:       LLM06
content_types:   [response]
action:          block
detector:
  type:    regex
  pattern: "(?i)(?:mrn[\s:]+[A-Za-z0-9\-]{4,}|..."
test_cases:
  should_match:
    - "Patient diagnosis: F32.0. MRN: P-447291"
  should_not_match:
    - "Please enter the patient ID in the system"
