Rules Architecture — Two-Layer Model
🔴 Substrate Rules — Always-On
Four rules fire unconditionally for every session on every AgentOS. No plugin can disable them. No operator override. They represent the minimum security boundary of the microVM substrate.
exfiltration-correlation → Aegis E2 (fleet)
host-network-escape-attempt → Cerberus E5 (fleet)
rootfs-write-attempt Permanent (session only)
vsock-bypass-attempt Permanent (session only)
🟢 Opt-In Rules — Per Session
Plugins register these at spawn time via POST /anomalies/configure-session. The plugin declares which protections it wants. Designed for community contribution — add new behavioral patterns without touching substrate code.
prompt-injection-marker LLM01 coverage
tap-leak-attempt LLM07 coverage
💡 Community rules in v0.2 wishlist:
memory-scrape-attempt · slow-exfil-drip · credential-echo · model-weight-read
📐 Rule migration path (ADR EMPIRE-475): Substrate rules that need cross-session context migrate to fleet-level components as they ship (exfil → Aegis E2, network-escape → Cerberus E5). Session layer remains the fast-path enforcer — 200ms budget cannot wait for a fleet round-trip.
